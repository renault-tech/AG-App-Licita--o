## D — Auto-save Universal

### O que existe
`editor-parecer.tsx` tem indicador de auto-save local (`salvando...` / `salvo` / `salvo automaticamente`). Os demais editores (DFD, ETP, TR, Riscos, Edital) não têm.

### Design

**Hook `useAutoSave`** em `src/hooks/use-auto-save.ts`:
- Recebe: `value: string`, `onSave: (v: string) => Promise<void>`, `delay = 1500`
- Estados internos: `'idle' | 'dirty' | 'saving' | 'saved' | 'error'`
- Lógica: quando `value` muda, estado vira `dirty` e agenda debounce de `delay`ms; ao disparar, chama `onSave`, aguarda, vira `saved`; se falhar, vira `error`
- Expõe: `{ status, lastSavedAt: Date | null, retrySave: () => void }`

**Componente `AutoSaveIndicator`** em `src/components/licita/auto-save-indicator.tsx`:
- Props: `status`, `lastSavedAt`, `onRetry`
- Renderiza (texto apenas, sem ícone pesado):
  - `idle`: nada
  - `dirty`: "Alterações pendentes..."
  - `saving`: spinner + "Salvando..."
  - `saved`: checkmark + "Salvo às HH:MM"
  - `error`: "Erro ao salvar — tentar novamente" (link clicável)
- Posicionamento: canto superior direito do card de cada editor, junto ao botão de salvar manual existente

**beforeunload** em cada editor Client Component:
```ts
useEffect(() => {
  const handler = (e: BeforeUnloadEvent) => {
    if (status === 'dirty' || status === 'saving') {
      e.preventDefault()
      e.returnValue = ''
    }
  }
  window.addEventListener('beforeunload', handler)
  return () => window.removeEventListener('beforeunload', handler)
}, [status])
```

**Arquivos a modificar:**
- `src/app/(dashboard)/processos/[id]/dfd/editor-dfd.tsx`
- `src/app/(dashboard)/processos/[id]/etp/page.tsx` (editor inline)
- `src/app/(dashboard)/processos/[id]/tr/page.tsx`
- `src/app/(dashboard)/processos/[id]/riscos/editor-riscos.tsx`
- `src/app/(dashboard)/processos/[id]/edital/page.tsx`
- Normalizar `editor-parecer.tsx` para usar o mesmo hook (remover lógica duplicada)

## B — Status Real das Etapas

### O que existe
O nav no layout do processo (`processos/[id]/layout.tsx`) determina `isConcluida` apenas pelo índice relativo à etapa ativa (baseado na URL). Não consulta o banco.

### Design

**Server Action `obterStatusEtapas`** em `src/lib/actions/processo.ts` (função nova):
```ts
type StatusEtapa = 'nao_iniciado' | 'rascunho' | 'em_revisao' | 'assinado' | 'devolvido'
type EtapaStatus = { slug: string; status: StatusEtapa; updated_at: string | null }

async function obterStatusEtapas(processoId: string): Promise<EtapaStatus[]>
```

Faz um `Promise.all` consultando cada tabela de documento pelo `processo_id`:
- `dfd`, `etp`, `termo_referencia` (slug: `tr`), `mapa_riscos` (slug: `riscos`), `edital`, `declaracoes` (slug: `declaracao`), `oficios` (slug: `oficio`), `pareceres` (slug: `parecer`)
- Campos selecionados: `status`, `updated_at`
- Se não encontrar registro: `nao_iniciado`
- `autorizacao` e `publicacao` usam suas tabelas próprias (`autorizacoes`, `publicacoes`)

**Integração no layout:**
- `processos/[id]/layout.tsx` chama `obterStatusEtapas` em paralelo com as demais queries já existentes
- Passa `etapasStatus: EtapaStatus[]` para o componente de nav
- O nav usa `etapasStatus.find(e => e.slug === etapa.slug)?.status` para determinar o ícone:
  - `nao_iniciado`: círculo cinza vazio
  - `rascunho` / `em_revisao`: círculo azul com ícone da etapa (igual ao atual para etapa ativa)
  - `assinado`: círculo verde com checkmark (igual ao atual `isConcluida`)
  - `devolvido`: círculo vermelho/âmbar com ícone de retorno
- Tooltip no círculo: "Última edição: DD/MM/YYYY HH:MM" ou "Não iniciado"

**Mapeamento slug → tabela:**
```
dfd           → dfd           (campos: status, updated_at)
cotacao       → cotacoes      (sem campo status — usar: existe linha? 'rascunho' : 'nao_iniciado'; usa updated_at)
etp           → etp           (campos: status, updated_at)
tr            → termo_referencia (campos: status, updated_at)
riscos        → mapa_riscos   (campos: status, updated_at)
edital        → edital        (campos: status, updated_at)
declaracao    → declaracoes   (campos: status, updated_at)
oficio        → oficios       (campos: status, updated_at)
revisao       → (sem tabela — inferir do campo fase_atual do processo)
parecer       → pareceres     (campos: status, updated_at)
autorizacao   → autorizacoes  (campos: status, updated_at)
publicacao    → publicacoes   (campos: status, updated_at)
```

## C — Busca + Paginação na Lista de Processos

### O que existe
`processos/page.tsx` é Server Component, aceita `searchParams.status` e `searchParams.fase`. Sem busca por texto, sem paginação — lista todos os processos da org em ordem cronológica.

### Design

**SearchParams adicionais:** `q` (string) e `page` (número, default 1)

**Busca server-side:**
```ts
// Sanitizar antes de usar em ilike
const qSafe = q.replace(/[%_;\\]/g, '')
if (qSafe) {
  query = query.or(`objeto.ilike.%${qSafe}%,numero_processo.ilike.%${qSafe}%`)
}
```

**Paginação:**
- `PAGE_SIZE = 20`
- `.range((page - 1) * PAGE_SIZE, page * PAGE_SIZE - 1)`
- Query separada para total: `.count()` com os mesmos filtros

**Componente `BuscaProcessos`** — Client Component em `src/app/(dashboard)/processos/busca-processos.tsx`:
- Input controlado com debounce de 300ms via `setTimeout`/`clearTimeout`
- Ao mudar, chama `router.push('/processos?q=<valor>&page=1')` preservando demais searchParams
- Lê valor inicial do searchParam `q` para preencher o campo ao voltar/recarregar

**Componente `PaginacaoProcessos`** — Server Component em `src/app/(dashboard)/processos/paginacao-processos.tsx`:
- Recebe `total`, `page`, `pageSize`, `q` (para manter na URL)
- Renderiza: "Mostrando X-Y de Z processos" + botões Anterior/Próximo como `<Link>` com href calculado
- Não usa estado client-side

**Ajuste na página:**
- Adicionar `BuscaProcessos` acima da lista de filtros existentes
- Adicionar `PaginacaoProcessos` abaixo da lista
- Mover query de contagem para junto da query principal

## A — Empty States

### Design

**Componente `EmptyState`** em `src/components/licita/empty-state.tsx`:
```tsx
interface EmptyStateProps {
  icon: LucideIcon
  titulo: string
  descricao: string
  cta?: { label: string; href?: string; onClick?: () => void }
}
```
- Layout: coluna centralizada, ícone 48px em círculo cinza, título sm/semibold, descrição xs/muted, botão CTA opcional
- Sem ilustração SVG complexa — ícone Lucide estilizado

**Aplicações:**

| Tela | Condição | Título | CTA |
|------|----------|--------|-----|
| `processos/page.tsx` | `processos.length === 0` | "Nenhum processo encontrado" | "Novo Processo" → `/processos/novo` |
| `processos/page.tsx` (com filtro) | lista vazia + filtro ativo | "Nenhum resultado para esse filtro" | "Limpar filtros" → `/processos` |
| `cotacao/cotacao-form.tsx` | sem itens na cotação | "Nenhum item adicionado" | "Adicionar item" (scroll/focus) |
| `procuradoria/page.tsx` | sem pareceres pendentes | "Nenhum parecer pendente" | — |
| `admin/organizacoes/page.tsx` | sem orgs pendentes | "Nenhuma prefeitura aguardando aprovação" | — |
