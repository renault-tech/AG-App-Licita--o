Layout da Fila de Pareceres

Como o procurador acessa e trabalha os pareceres pendentes?

Contexto atual

Hoje o procurador só chega ao parecer via link direto no processo. Não há visão consolidada das pendências. O dashboard já mostra uma lista simples, mas não é uma área de trabalho dedicada.

A

Rota dedicada /procuradoria

Área separada da navbar com fila de pareceres pendentes, histórico e filtros. O procurador tem seu "escritório" próprio, desacoplado do fluxo de processos.

Pontos fortes

  • Visão completa de todas as pendências
  • Filtros e busca independentes
  • Histórico de pareceres emitidos

Contra

  • Mais código novo
  • Precisa de link extra na navbar
B

Página dentro de /processos

Uma rota /processos?papel=procurador que filtra a lista de processos para mostrar apenas os que estão na etapa de parecer. Reutiliza o componente de lista existente.

Pontos fortes

  • Menos código — reusa o existente
  • Sem mudança na navbar

Contra

  • UX genérica, não focada no procurador
  • Sem histórico consolidado
C

Expandir o Dashboard do Procurador

O dashboard já tem uma seção para o procurador. Torná-la mais completa: fila com filtros, contador de pendências, atalho para abrir cada parecer — tudo na mesma página.

Pontos fortes

  • Sem nova rota
  • Tudo em um lugar

Contra

  • Dashboard fica pesado
  • Não é uma área de trabalho real
